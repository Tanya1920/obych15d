Порядок ежедневной работы,выполнение заданий.

1.Проверка обновлний
git pull
pull (лучше) делать на каждую ветку в отдельности
2. Проверка состояний
git status
3. Перед началом работы нужно перейти в свою ветку
создание новой ветки
git checkout master
git checkout -b
4.  Работа в новой ветке
5.  Сохранение работы
git add .
git commit -a -m "Комментарий"
6. Отправка изменений на гитхаб
первый раз
git push --set-upstream origin <имя ветки>
если повторно,то 
если в эту ветку уже пушили, команда
git push
